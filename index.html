Comment ca fonctionne

FRAGMENT applique un chiffrement XOR one-time pad avec un masque aleatoire CSPRNG. Deux fragments independants, inutiles l'un sans l'autre.

Generation mathematique

Stockage separe

Fragment A → support numerique securise. Fragment B → support physique (impression, notaire). Les deux sont necessaires.

Non, jamais. Tout le traitement se deroule dans votre navigateur. Aucun appel reseau. Verifiez avec F12 — aucune requete pendant une fragmentation.

Que se passe-t-il si je perds un fragment ?+
Le secret est definitivement perdu. Sans les deux fragments, aucune reconstruction n'est possible meme avec une puissance de calcul infinie.
L'algorithme XOR est-il vraiment securise ?+
Oui, sous trois conditions : masque aussi long que le message, utilise une seule fois, genere aleatoirement. FRAGMENT respecte ces trois conditions. Prouve parfait par Claude Shannon en 1949.

Comment securiser votre seed phrase en 2026

Votre seed phrase est la cle absolue de vos actifs crypto. Si quelqu'un la trouve, il a acces a tout.

Le protocole FRAGMENT

1. Ouvrez FRAGMENT ELITE deconnecte du reseau. Saisissez vos mots dans l'ordre.

2. Cliquez FRAGMENTER. Ajoutez un indice. Imprimez le kit.

3. Fragment A → coffre bancaire. Fragment B → notaire. Jamais ensemble.

Cryptographie · XOR

Le chiffrement XOR explique simplement

Si A XOR B = C, alors C XOR B = A. C'est le principe de FRAGMENT : A XOR B = Secret original.

La preuve de Shannon (1949)

Le XOR one-time pad offre une securite parfaite si la cle est aussi longue que le message, generee aleatoirement, jamais reutilisee.

Pratique · Dual-Key

Protocole dual-key pour les nuls

1. A et B ne doivent jamais etre au meme endroit.

2. Support numerique pour l'un, physique pour l'autre.

3. Donnez B a quelqu'un de confiance avec les instructions.

4. Testez la reconstitution une fois par an.
